# Rules Gate Bot

Telegram-бот для групп: встречает новичков, показывает правила и не даёт писать,
пока не нажата кнопка согласия. Опционально шлёт аудит-лог событий чата
(join / ban / kick / leave / согласие с правилами) в отдельный лог-чат.

Написан на [aiogram 3.x](https://docs.aiogram.dev/).

## Возможности

- 👋 На вход новый участник получает мут и приветствие с правилами и inline-кнопкой.
- 🔓 Мут снимается только после нажатия кнопки самим этим участником.
- ⏳ Приветствие автоматически удаляется через настраиваемый TTL (по умолчанию 300 сек, `0` — не удалять).
- ✍️ Правила, текст кнопки и TTL — настраиваются командами, отдельно для каждого чата.
- 🗒 Аудит событий чата в отдельный лог-чат (опционально).

## Установка

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env  # вписать BOT_TOKEN
```

## Запуск

```bash
export $(grep -v '^#' .env | xargs) && python bot.py
```

Бот работает в режиме long polling. Веб-хук намеренно сбрасывается на старте.

## Настройка бота в BotFather

1. Создать бота через [@BotFather](https://t.me/BotFather), получить токен.
2. `/setprivacy` → **Disable** — иначе бот не увидит события `chat_member`
   и не будет знать о новых участниках.
3. Добавить бота в группу и выдать права администратора:
   - **Restrict members** — для мута/размута новичков.
   - **Delete messages** — чтобы убирать приветствие после согласия / по TTL.
   - **Ban users** — если планируешь логировать баны/кики.
4. Убедиться, что группа — супергруппа (Telegram сам конвертирует при выдаче
   прав админам). Обычные группы не поддерживают `restrict_chat_member`.

Без этих прав вызовы бота молча провалятся: ошибки Telegram подавляются, чтобы
не спамить чат.

## Конфигурация

`.env`:

| Переменная    | Обязательна | Значение по умолчанию | Описание                                   |
| ------------- | ----------- | --------------------- | ------------------------------------------ |
| `BOT_TOKEN`   | да          | —                     | Токен от BotFather                         |
| `WELCOME_TTL` | нет         | `300`                 | Дефолтный TTL приветствия в секундах       |

Дефолтный текст правил и надпись на кнопке лежат в `config.py`
(`RULES_TEXT`, `BUTTON_TEXT`).

## Команды

Все команды `/set*`, `/reset*`, `/unset*` доступны только администраторам чата.
Команды `/rules`, `/ttl`, `/button`, `/log` — всем.

### Правила

| Команда              | Что делает                                                                              |
| -------------------- | --------------------------------------------------------------------------------------- |
| `/rules`             | Показать текущие правила чата                                                           |
| `/setrules <текст>`  | Задать правила. Поддерживает HTML. Можно `/setrules` реплаем — возьмётся текст ответа   |
| `/resetrules`        | Сбросить к дефолту из `config.py`                                                       |

### Кнопка

| Команда               | Что делает                                                       |
| --------------------- | ---------------------------------------------------------------- |
| `/button`             | Показать текущий текст кнопки                                    |
| `/setbutton <текст>`  | Задать (до 64 символов — лимит Telegram inline-кнопки)           |
| `/resetbutton`        | Сбросить к дефолту `✅ Я согласен с правилами`                   |

### TTL приветствия

| Команда              | Что делает                                                        |
| -------------------- | ----------------------------------------------------------------- |
| `/ttl`               | Показать текущий TTL                                              |
| `/setttl <секунды>`  | Задать (`0..86400`). `0` — не удалять приветствие автоматически   |
| `/resetttl`          | Сбросить к дефолту из `WELCOME_TTL`                               |

### Кик по TTL

| Команда              | Что делает                                                                                 |
| -------------------- | ------------------------------------------------------------------------------------------ |
| `/kick`              | Показать текущий режим                                                                     |
| `/setkick on\|off`   | `on` (дефолт) — кикать не согласившихся по TTL; `off` — просто размутить и оставить в чате |
| `/resetkick`         | Сбросить к дефолту (`on`)                                                                  |

### Лог-чат

| Команда              | Что делает                                                              |
| -------------------- | ----------------------------------------------------------------------- |
| `/log`               | Показать `chat_id` текущего лог-чата                                    |
| `/setlog <chat_id>`  | Явный id, или `/setlog` реплаем на пересланное из лог-чата сообщение    |
| `/unsetlog`          | Отключить логирование                                                   |

Как настроить лог-чат:

1. Создать отдельный чат (можно приватный), добавить туда бота.
2. В основном чате выполнить `/setlog <id_лог_чата>` — бот тут же напишет в
   лог-чат «этот чат назначен лог-чатом для …». Если не смог — команда откажет.
3. Альтернатива: переслать любое сообщение из лог-чата в основной чат и
   ответить на него `/setlog` — id подхватится из `forward_from_chat` (работает
   для публичных каналов/групп; для приватных проще указать id вручную).

Какие события пишутся в лог-чат:

| Иконка | Событие                                                       |
| ------ | ------------------------------------------------------------- |
| `➕`   | Новый участник (плюс кто добавил, если это админ)             |
| `✅`   | Участник принял правила (кнопка нажата)                       |
| `🚫`   | Бан (кто и кого)                                              |
| `♻️`   | Разбан                                                        |
| `👋`   | Участник сам вышел                                            |
| `👢`   | Kick (админ удалил без бана)                                  |

## Хранение состояния

Все настройки чатов (правила, кнопка, TTL, log-chat) лежат в `chats.json`
рядом с `bot.py`:

```json
{
  "-1001234567890": {
    "rules": "<b>Правила</b> ...",
    "button": "Согласен",
    "ttl": 120,
    "log_chat": -1009876543210
  }
}
```

Никаких БД и миграций — простой JSON, читается/пишется под `asyncio.Lock`.
Файл создаётся автоматически при первом изменении.
